# 模型格式转换示例

本目录覆盖工具当前真实支持的六类输入和七类输出。

| 编号 | 输入类型 | 主要验证内容 |
|---|---|---|
| 01 | OBJ 封闭块体 | 多边形面、工程大坐标、七格式输出 |
| 02 | ASCII STL | 三角表面和顶点焊接 |
| 03 | ASCII PLY | 顶点颜色与岩性编码属性旁车 |
| 04 | Legacy ASCII VTK | 数值模拟表面网格 |
| 05 | GeoJSON Polygon | GIS 面要素及面属性 |
| 06 | CSV | 钻孔与监测点、逐点属性 |
| 07 | 非法 OBJ | 越界面索引和失败原子性 |
| 08 | GeoJSON LineString | 断层线、地层边界和线属性 |

每次成功转换都会生成：

1. 目标格式主文件；
2. `conversion_manifest.json`，记录转换 ID、源/成果 SHA-256、坐标变换、回读校验和显式信息损失；
3. 必要时生成 `*.attributes.json` 属性旁车，保存目标主格式不能完整承载的拓扑、属性和元数据。

点数据没有表面面片，因此不能直接输出 STL 或 IFC；工具会返回明确错误，不生成伪成果。涉及真实 CRS 投影转换时，应补充源 CRS、目标 CRS 并接入 pyproj/GDAL，不能仅凭坐标值猜测。
