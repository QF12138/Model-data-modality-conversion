数据预处理与质量清洗模块——软件内运行示例
================================================

一、放置位置
将 preprocessing_examples 文件夹整体放到项目的 source 目录下：

项目根目录/
├─ main.py
├─ function/
└─ source/
   └─ preprocessing_examples/
      ├─ 01_钻孔数据_综合问题.csv
      ├─ 02_剖面数据_缺少终点Y字段.csv
      ├─ 03_点云数据_重复缺失与高程异常.csv
      ├─ 04_栅格像元_缺失重复与异常.csv
      ├─ 05_矢量点要素_属性缺失与重复.geojson
      ├─ 06_网格模型_重复顶点非法面与开放边.obj
      └─ 07_正常地质样本_对照.csv

二、软件内运行
1. 启动主程序。
2. 左侧选择“数据准备与标准化 → 数据预处理与质量清洗”。
3. 点击“添加数据”。
4. 进入 source/preprocessing_examples，选择一个或多个示例文件。
5. 点击“执行数据清洗”。
6. 在字段概览、清洗结论和问题清单中查看结果。

三、参数配置建议
1. 钻孔数据单独测试：
   字段必填规则：
   borehole_id,x,y,z,depth,lithology

   重复识别键：
   borehole_id

   异常值阈值：
   3.0

2. 剖面数据单独测试：
   字段必填规则：
   剖面编号,起点X,起点Y,终点X,终点Y

   重复识别键：
   剖面编号

   该文件故意没有“终点Y”列，应报告缺少必填字段。

3. 点云数据单独测试：
   字段必填规则：
   point_id,x,y,z

   重复识别键：
   x,y,z

   异常值阈值：
   3.0

4. 栅格数据单独测试：
   字段必填规则：
   cell_id,x,y,pixel_value

   重复识别键：
   x,y

5. 混合批量测试：
   多种数据类型一起选择时，建议暂时将“字段必填规则”和“重复识别键”留空，
   因为当前界面对整批文件使用同一套全局规则；不同数据类型的字段结构并不相同。

四、每个文件的用途
01 钻孔数据：
- lithology、water_level、remark 存在缺失值；
- 含一条完整重复记录；
- depth=1600 为明显异常值；
- dip_angle 会触发角度单位规范化提示。

02 剖面数据：
- 故意缺少“终点Y”字段；
- 含重复剖面记录；
- 需要在参数配置中填写必填字段规则，才能显示缺字段问题。

03 点云数据：
- classification 存在缺失；
- 含重复点；
- 最后一个点的 z=9999，为明显异常高程。

04 栅格像元：
- pixel_value 和 quality_flag 存在缺失；
- 含重复像元；
- pixel_value=5000 为异常像元。

05 GeoJSON：
- 部分要素缺少岩性或密度；
- 含重复要素；
- dip_angle 用于测试角度单位提示。

06 OBJ：
- 含重复顶点；
- 一个面引用不存在的第99号顶点；
- 缺少顶面，属于开放网格；
- 含一个远离主体的异常顶点。

07 正常对照：
- 无缺失、无重复、无明显异常；
- 用于与问题数据的评分进行对比。

注意：
当前 build_preprocessing_report 主要生成质量检查报告，不会直接修改源文件。
